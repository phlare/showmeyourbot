Add Link Protection in Streamer Bot

December 27, 2023
Streamer.bot

Want to add some basic link protection to your Streamer bot setup?

This action will read each chat message and delete any that contain links.

  • Establish a whitelist of domains that are allowed at any time regardless of permissions.
  • Configure the message sent to chat while a link is deleted.
  • Permissions are configurable so that Bots, Moderators, or VIPs can all be permitted to post links regardless of domain.

Installation Instructions

These instructions assume you already have streamer bot installed and connected to your Twitch account. Go read this guide to learn how to set up streamer bot intially.

  1. Copy the Import Code below (triple-click to select easily)
  2. Click Import at the top of Streamer bot in order to open the import dialog
  3. Paste the code you copied earlier into the "Import String" box
  4. Make sure Link Protection shows up as an action
  5. Click Import at the bottom

    Configuring Link Protection

    1. Open the Link Protection action that you just imported
    2. change phlare_bot to your own bot name in the "if" statement (this isn't strictly necessary if you have allowBots set to True and your bot is part of your bot Group (see #5 and #6) but it's a good failsafe.
    3. Check out the %whitelist% argument and add or remove whitelisted domains as desired. leave a single space between domains. Streamer bot allows links from these domains, regardless of permissions. subdomains should be included with any base domain whitelisted, so no need to add both twitch.tv and clips.twitch.tv
    4. Set the warning you wish to send to chat when the bot deletes a link. You may use variables here if you wish.

      defaults message: hey %user%, no links please!
    5. Set the name of your Bot Group in the %botGroup% variable. See my video on group permissions here to learn how to set up a bot group: https://youtu.be/MdlmfH92vOg
    6. Set Permissions:
      The last section has three arguments - %allowVips%, %allowMods%, and %allowBots%.
      Set these to True or False as needed.
      Setting one of these to True will allow that role to bypass the link protection rules.
      Setting one of these to False will prevent that role from posting links outside of the whitelisted domains.
